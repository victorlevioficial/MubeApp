# Security Policy

## Supported Versions

Security reports are accepted for the active codebase on the `main` and `develop` branches.

## Reporting a Vulnerability

If you discover a security issue, do not open a public GitHub issue.

Please report it by email to `suporte@mube.app` with the subject:

`[SECURITY] Short description of the issue`

Include, when possible:

- affected area or file
- reproduction steps
- impact assessment
- proof of concept or screenshots
- suggested remediation, if available

## Response Process

- Initial acknowledgement target: up to 5 business days
- Follow-up updates: as investigation progresses
- Public disclosure should wait until the issue is validated and a fix or mitigation is available
